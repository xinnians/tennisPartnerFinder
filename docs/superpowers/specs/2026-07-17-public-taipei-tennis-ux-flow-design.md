# 台北網球公開 MVP UX 流程規格

日期：2026-07-17
狀態：創辦人已在對話中逐項確認；待審閱本文後，才更新實作計畫

## 1. 目的與適用範圍

本文件為公開台北市網球 MVP 的互動與資訊架構規格。它補足
[公開 MVP 產品設計](2026-07-17-taipei-tennis-public-mvp-design.md) 的 UI/UX
決策，並優先於既有實作計畫中與本文件衝突的首頁、公開球局詳情、開局表單、
我的球局與登入回跳描述。

產品主流程不變：

~~~text
看見附近球局 → 申請加入 → 主揪核准／婉拒 → 核准後交換 LINE → 回報打成
~~~

本文件不新增訂場、付款、聊天室、通知、評價、候補、其他運動或其他城市。

## 2. 已確認的 UX 決策

| 主題 | 決策 |
|---|---|
| 首頁主體 | 地圖優先，不以列表作為第一屏 |
| 定位請求 | 不自動跳系統權限；使用者主動按「使用我的位置」才請求 |
| 定位後視野 | 以使用者為中心約 5 公里的初始地圖範圍，可自由拖曳、縮放、重新置中 |
| 首頁資訊密度 | 採「地圖＋收合的附近球局抽屜」；不是純地圖，也不是全螢幕列表 |
| 公開詳情 | 顯示主揪暱稱與 NTRP，但不顯示 LINE、真實姓名、個人檔案連結或評價 |
| 開球局 | 單一手機底部表單；必要欄位先、選填後 |
| 我的球局 | 依下一步行動分為「需要你處理／即將打球／過去紀錄」 |
| 登入與補資料 | 保留原始加入意圖，完成後回原球局的加入確認，不自動送出 |

## 3. 隱私與公開資料的 UX 邊界

### 3.1 公開球局中可揭露的資料

公開的 session discovery 與球局詳情可顯示：

- 球場、行政區、開始時間、打法、程度範圍、剩餘缺額與備註。
- 主揪的公開暱稱與 NTRP。
- 中性提示「主揪已完成基本資料」。

它不可顯示：

- LINE ID、手機、email、真實姓名或可連至公開個人頁的 profile ID。
- 主揪的常打球場、固定時段、歷史出席率、評價或其他未經設計的個人資料。
- 任一申請者或其他已核准客人的資料。

因此，公開 discovery 需要明確、窄化的欄位：

~~~text
session_id, court fields, start_at, play_type, NTRP range,
slots_total, slots_remaining, notes, status,
host_nickname, host_ntrp, host_profile_complete
~~~

host_nickname 和 host_ntrp 是刻意公開的展示欄位，不是讓前端取得完整
profiles 資料的理由。資料庫 view 不提供 host_profile_id、LINE 或其他 profile
欄位；LINE 仍只由 accepted host/guest 配對的 session_contacts view 讀取。

### 3.2 個人檔案與同意

個人檔案中的暱稱欄位文案改為「公開暱稱」，並附註：

> 開球局後，這個暱稱與你的 NTRP 會顯示給瀏覽該球局的人；LINE ID 只會在你核准加入者後顯示。

建立球局表單在送出按鈕前再重申一次同一件事。這是建立公開球局時的明確告知，
不是以隱藏設定取代同意。

### 3.3 位置資料

產品資料模型不儲存使用者的目前位置。使用者位置只用來決定當下地圖中心與
目前地圖範圍的 discovery 查詢；不寫入 profiles、sessions、reports 或分析事件。

## 4. 首頁：地圖與附近球局

### 4.1 初始狀態

使用者從 LINE／Facebook 連結開啟網站時：

1. 顯示台北市預設地圖、現有公開球局圖釘與球場底圖釘。
2. 顯示「使用我的位置」按鈕，但不要求定位權限。
3. 顯示「開球局」主要動作與時間、程度、打法篩選。
4. 地圖下方顯示收合抽屜：
   - 尚未定位時：這個地圖範圍內 N 場可加入。
   - 已定位時：附近 N 場可加入。
   - 顯示最近一場的時間、球場、打法與缺額摘要。

收合抽屜本身可點擊或上滑展開。展開後顯示目前地圖範圍內、依開始時間排序的
球局列表。使用者不需先點圖釘才能確定有沒有可加入的局。

### 4.2 定位後狀態

使用者按「使用我的位置」後：

1. 才呼叫瀏覽器定位 API。
2. 成功時，以使用者位置為中心顯示約 5 公里的初始視野，並用「你」標記當前位置。
3. 收合抽屜改用「附近」文案，展開列表優先依距離、再依開始時間排序。
4. 使用者可移動地圖；此後抽屜代表目前地圖範圍，不把 5 公里當成硬性篩選。
5. 使用者再次按定位按鈕時，回到最新位置。

若使用者拒絕或裝置無法定位，顯示非阻斷訊息：

> 無法取得位置；你仍可移動地圖或依球場尋找球局。

地圖與抽屜持續可用，且不反覆要求權限。

### 4.3 圖釘、抽屜與空狀態

- 每個公開球局是可點擊的「局」圖釘；同一球場多局時使用數量圖釘。
- 點擊圖釘或展開列表的球局列，都開啟同一個球局詳情底部 sheet。
- 球場底圖釘仍可開啟該球場抽屜，顯示當前篩選下的球局。
- discovery 載入中，地圖與球場底圖釘可先操作，抽屜顯示載入狀態。
- 目前範圍沒有球局時，抽屜誠實顯示：

~~~text
這個範圍暫時沒有可加入的球局
[擴大地圖範圍] [開第一局]
~~~

不可用假球局或舊球局填補空畫面。

## 5. 球局詳情與申請加入

### 5.1 詳情 sheet 的資訊順序

點選球局後，手機底部 sheet 依下列順序呈現：

1. 球場名稱與行政區。
2. 開始日期／時間（台北時間）。
3. 打法、程度範圍、還缺幾位。
4. 主揪資訊：公開暱稱、NTRP、主揪已完成基本資料。
5. 備註。
6. 主要動作。

主要動作依觀看者狀態改變：

| 狀態 | 主要動作 |
|---|---|
| 匿名或未完成資料 | 申請加入，後續進登入／補資料流程 |
| 已完成資料、尚未申請 | 申請加入 |
| 已申請 | 等待主揪回覆與撤回申請 |
| 已核准 | 查看聯絡方式 |
| 額滿 | 已額滿，不可申請 |
| 取消、過期或開始後 | 不可申請，顯示原因 |

公開詳情不呈現快速約球、站內聊天、評價、其他客人或 LINE。

### 5.2 申請確認

第一次按「申請加入」不直接寫入資料庫，而是開啟一張確認卡，重複顯示球局摘要並
提供最後的「確認申請加入」。這能避免地圖誤觸，也讓登入或補資料回跳有明確的目的地。

成功後顯示：

> 已送出申請，等待主揪回覆。

不承諾即時通知；使用者可在「我的球局」查看狀態。

## 6. 開球局：單一底部表單

開球局使用一張可捲動的手機底部表單，不拆成多步驟。

### 6.1 必填欄位

依填寫順序：

1. 球場（僅台北市有效網球場）。
2. 開始時間（台北時間）。
3. 打法（單打、雙打、對拉、練球）。
4. 缺額（1–3 位）。

### 6.2 選填欄位

- 適合程度範圍。
- 備註，最多 500 字。

必填內容放在首屏；選填內容置於其後，避免讓已持有場地與時間的主揪因長表單而放棄。
送出時驗證未來時間、合法程度範圍、缺額與備註長度；錯誤就地顯示，不關閉表單。

送出成功後：

1. 關閉表單。
2. 重新整理目前地圖。
3. 導到「我的球局」的即將打球區塊。
4. 聚焦剛建立的球局卡。

## 7. 我的球局：行動導向頁面

「我的球局」不以資料庫角色作為三個平行分頁，而以使用者下一步要做什麼排序。

### 7.1 需要你處理

僅顯示需要立即處理的事項：

- 主揪的待審核申請：顯示申請者暱稱、NTRP、打法、常打球場，以及接受／婉拒。
- 已申請者的可撤回申請不必製造 badge，但可在卡上顯示撤回動作。

底部導覽的「我的球局」只為主揪待審核申請顯示數字 badge。

### 7.2 即將打球

將自己主揪的未開始球局與已核准參加的未開始球局合併，依開始時間排序。每張卡
明確標示「我是主揪」或「已核准加入」，並只露出此刻合理的下一步：

- 主揪：查看申請、取消球局。
- 已核准客人：查看 accepted-only 聯絡方式、開始前退出。
- 開始後：主揪可回報「打成了」；已核准參加者可確認到場。

### 7.3 過去紀錄

已打成、取消、過期、婉拒與撤回的球局留在此區，附狀態原因，但不混入需要立即處理
的畫面。

## 8. 登入、補資料與回跳

### 8.1 加入意圖

使用者開始加入流程時，只保存最小意圖：

~~~json
{ "action": "join", "sessionId": 123 }
~~~

建立球局時保存：

~~~json
{ "action": "create" }
~~~

意圖不包含 LINE、定位、草稿備註、表單內容或帳號 token。

### 8.2 流程

1. 匿名使用者點「申請加入」，開啟登入，不失去目標球局。
2. OAuth 成功後，重新讀取該球局，回到加入確認卡。
3. 已登入但資料不完整時，導到個人檔案頁；頁首固定顯示：

~~~text
完成後將回到：{球場}・{開始時間}
~~~

4. 個人檔案儲存成功後，回到同一張加入確認卡。
5. 使用者需自行按最後一次「確認申請加入」；系統不自動送出。
6. 若球局在流程中已額滿、取消或過期，清除意圖、說明原因、回到附近地圖。
7. 關閉登入、登出或主動取消時，清除意圖。

## 9. 手機、可近用性與錯誤恢復

- 首發以 390px 寬度手機優先；地圖、收合抽屜與所有底部 sheet 不得超出視窗。
- 抽屜、球局詳情、加入確認與開球局表單使用語意 dialog，支援關閉、Esc、焦點回復與螢幕閱讀器標籤。
- 篩選與定位操作有即時 loading、empty、error 與 retry 回饋。
- 地圖金鑰或載入失敗時，使用抽屜的可展開球局列表繼續探索，而不是展示給一般使用者的開發設定說明。
- 所有非成功動作保留原本上下文；例如加入被後端拒絕時，詳情 sheet 保持開啟，重整後呈現最新狀態。

## 10. 對現有實作計畫的必要修正

[公開 MVP 實作計畫](../plans/2026-07-17-public-taipei-tennis-session-mvp.md) 在開始實作前必須依本規格調整：

1. public SessionSummary 與 session_discovery 加入 hostNickname、hostNtrp、hostProfileComplete 三個明確公開欄位，但不加入 hostProfileId 或 LINE。
2. 首頁取消「地圖／列表」並列的全螢幕 segmented 預設；改為地圖主體與可展開的附近球局抽屜。列表仍存在於抽屜內。
3. 位置權限改為使用者主動觸發，初始定位視野為約 5 公里，且無位置持久化。
4. 球局詳情加入主揪公開暱稱與 NTRP，並新增建局前的公開揭露告知。
5. 開球局維持單一表單，不加入多步精靈。
6. 我的球局頁面由「我開的／已申請／已核准」改為「需要你處理／即將打球／過去紀錄」。
7. Playwright 與 pgTAP 更新公開欄位 allowlist、無 LINE 洩露、位置權限拒絕、收合抽屜、主揪資訊揭露、行動排序與登入回跳覆蓋。

## 11. UX 驗收條件

- 初次開啟不出現系統定位權限請求；按「使用我的位置」後才請求。
- 拒絕定位後仍可從地圖與抽屜找到球局。
- 定位成功後，地圖顯示約 5 公里的初始視野，且「附近」抽屜反映當前地圖範圍。
- 使用者在不點圖釘的情況下，從收合抽屜即可知道目前範圍是否有可加入球局。
- 公開詳情可看見主揪暱稱與 NTRP，但匿名 HTML、REST 回應與 marker 不含 LINE 或 profile ID。
- 開局者能在一張表單內完成必填資料並理解自己的暱稱、NTRP 會公開。
- 主揪有待審核申請時，「我的球局」優先顯示該工作與 badge。
- 完成登入或個人檔案後，使用者回到原球局的加入確認，不會意外送出申請。
- 額滿、取消、過期、網路失敗與地圖失敗都提供清楚下一步，沒有死路。

## 12. 非目標

- 不做使用者評分、回覆率、出席率、黑名單或可信度分數。
- 不自動保存、追蹤或分享使用者精確位置。
- 不公開 LINE，也不讓同場已核准的客人互相取得聯絡方式。
- 不為了冷啟動而生成假球局、假主揪或假附近清單。
