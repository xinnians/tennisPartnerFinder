# 台北網球生態研究卡（官方來源模板）

這份研究用來幫助選擇初期試點、球場引導與社群發布順序；它不是公開上線的替代品，也不會
用來推測個人常去球場或建立使用者名單。首輪目標是完成 **15 張可重複驗證的官方來源卡**。

## 研究原則

- 只採用管理單位、場館、學校、地方政府、運動中心或正式課程主辦方的公開頁面。
- 每一項可變資訊（訂場、開放時段、課程）要記錄來源 URL 與驗證日期；過期就標示待複核，
  不可假裝仍有效。
- 「有網球場」不等於「適合首發試點」。pilot suitability 要綜合交通、可訂性、公開活動、
  服務範圍與資料可信度判斷，並寫下理由。
- 不要爬取、轉貼、儲存或推論私人 LINE／Facebook 群組貼文、名單、聯絡方式或活動。
- 不要把未被官方／場館來源證實的「俱樂部包場」「常態佔場」「某群固定使用」說法寫成事實。
  若此類資訊影響產品決策，只能標為未驗證假設，並先向正式管理單位查證。

## 每張球場卡的必要欄位

```md
### [編號] [球場名稱]

- 行政區／交通：行政區、最近大眾運輸或官方交通說明。
- 訂場方式：官方預約系統、現場規則、費用資訊或「待官方確認」。
- 可驗證空檔：官方場地時段、預約頁可見 availability、或明確標示無公開資料。
- 固定活動／課程證據：官方公告的課程、聯賽、社區活動；沒有就寫「尚無官方證據」。
- 官方來源 URL：直接連到支持上述資料的頁面。
- 最後驗證日：YYYY-MM-DD。
- Pilot 適合度：高／中／低／待評估，以及一至兩句理由。
- 待確認項目：下一次應向哪個官方單位確認什麼。
```

## 使用流程

1. 先由 `data/courts.json` 的台北市 active courts 選候選，避免憑私人社群印象選點。
2. 對每座候選球場先填官方 URL 與驗證日期；找不到資料是可記錄的結果，不要補猜。
3. 用必要欄位寫成一張研究卡，再更新上方追蹤表。
4. 每 90 天或管理制度改變時重驗；若來源失效，將 status 改為待確認。
5. 只有「官方資料可重現」與「產品 funnel 有實際證據」同時成立時，才調整試點／社群策略。

## 不可做的研究捷徑

- 不抓取或保存私人社群貼文、截圖、使用者名稱、LINE ID、電話或 email。
- 不以搜尋結果摘要、轉述貼文或單一個人經驗當作官方 availability／訂場規則。
- 不根據未驗證的俱樂部佔場傳聞封鎖、排序或標記球場。
- 不把研究卡的資料直接轉成使用者資料、行銷名單或假球局。
